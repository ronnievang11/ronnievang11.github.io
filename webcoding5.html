UTF-8

This is very interesting to learn about. It is great to know and understand Unicode in how it work and being used for computers, especially when it was started off from writing on a piece of napkin. The ASCII table was a very unique set of binary system that is use for computers to be able to understand since computer can only understand numbers. It uses byte to create letters. Because ASCII was created and mainly use for English, other places such as Japan develop their own type. Mojibake was an encoding character for Japan to use for their language. The point of Mojibake is for Japanese email or usenet posting. Now we have UTF-8 that everyone is using, which stands for Unicode Transformation Format-8. It is also by far the best encoding of Unicode characters. Like the video explain, UTF-8 really solves all the problem with special characters, symbols, and different languages letters. UTF-8 is compatible with nul-terminated strings. No character in UTF-8 will have a nul, which is 0, byte when encoded. Their byte is a continuation that goes on and on. Just watching the video about how the byte continuation works in UTF-8 is amazing.
